# TV — FongMi Android TV/Phone Video Player

## Repository Purpose

An open-source Android video player app based on [CatVod](https://github.com/CatVodTVOfficial/CatVodTVJarLoader), supporting both **Android TV** (`leanback` flavor) and **phone** (`mobile` flavor) via product flavors. Content is extended through external JSON configuration (vod/live sources, spiders).

## Quick Start & Build

```bash
# Full debug build (all flavors)
./gradlew assembleDebug

# Release build for a specific flavor
./gradlew assembleLeanbackRelease   # TV版
./gradlew assembleMobileRelease     # 手机版

# Release APKs are automatically copied to Release/apk/ after build
```

- **Java**: sourceCompatibility/targetCompatibility = **Java 21**
- **Min SDK**: 24 (Android 7.0)
- **ABI targets**: `arm64-v8a`, `armeabi-v7a`
- **Signing**: Release builds require `storeFile`, `keyAlias`, `storePassword` in `local.properties`
- No lint/typecheck commands are defined; Gradle lint runs as part of build

## Architecture & Key Conventions

### Package layout (`app/src/main/java/com/fongmi/android/tv/`)

| Package | Purpose |
|---------|---------|
| `api/` | Network API clients |
| `bean/` | Data models/POJOs |
| `browse/` | Browse/explore UI logic |
| `db/` | Room database |
| `dlna/` | DLNA (UPnP) streaming |
| `event/` | EventBus events |
| `exception/` | Custom exceptions |
| `gson/` | Gson type adapters |
| `impl/` | Interface implementations |
| `model/` | View models / state holders |
| `playback/` | Playback service & controls |
| `player/` | Player (ExoPlayer/Media3) |
| `receiver/` | Broadcast receivers |
| `server/` | NanoHTTPD local HTTP server |
| `service/` | Android services |
| `setting/` | Settings & preferences |
| `ui/` | UI screens & fragments |
| `utils/` | Utility classes |

### Build Variants (2 dimensions)
- **Mode**: `leanback` (TV), `mobile` (phone)
- **ABI**: `arm64_v8a`, `armeabi_v7a`
- Output: `{mode}-{abi}.apk`

### Key Dependencies
- **Player**: Media3 (ExoPlayer) + FFmpeg
- **UI**: ViewBinding, AndroidX Leanback (TV), Material Design
- **Networking**: OkHttp, Gson
- **Database**: Room (schema → `app/schemas/`)
- **Event bus**: EventBus (index → `com.fongmi.android.tv.event.EventIndex`)
- **Image loading**: Glide
- **DLNA**: JUPnP
- **Local server**: NanoHTTPD

### Coding Conventions
- Java 21, no Kotlin in the main app
- ViewBinding enabled (use generated binding classes)
- EventBus for decoupled event passing (uses annotation processor for index)
- Room for persistence (export schema to `app/schemas/`)
- Gson for JSON serialization
- Product flavors for UI variants; `main/` sourceset for shared logic
- Do NOT add new module dependencies without updating `settings.gradle`

### Known Gotchas
1. **Release builds** require `local.properties` with signing config — debug builds do not
2. **Room schema changes** require deleting old schema files from `app/schemas/` or updating version
3. **EventBus index** is generated by annotation processor — after adding new event classes, rebuild
4. **ProGuard**: Release builds use `proguard-rules.pro` + `proguard-rules-media.pro` — test release builds before shipping
5. **Chaquopy (Python)**: Python code lives in `chaquo/src/main/python/`, pip requirements in `chaquo/requirements.txt`
6. **Module dependency chain**: `:app` → `:quickjs`, `:chaquo` → `:catvod` (catvod is the base library)
7. **Non-module directories** (`thunder/`, `tvbus/`, `hook/`, `jianpian/`, `zlive/`, `forcetech/`) contain engine implementations used by catvod — they are NOT Gradle submodules
8. **Remote control server** binds on port 9978–9998 (NanoHTTPD)
9. **EPG refresh** happens every 6 hours for live TV
10. **`.gitignore`** excludes `*.jks`, `*.properties`, `lib-*.aar`, `/Release`, `/docs`. `.github/workflows/` is tracked (CI), other `.github/` content is ignored.
11. **CI/CD**: GitHub Actions workflow at `.github/workflows/build.yml` — see [CI/CD](#cicd) below.

## CI/CD

GitHub Actions workflow at `.github/workflows/build.yml`:

- **Trigger**: Push/PR to `fongmi` branch, or manual dispatch
- **Build types**: `debug` (default on push/PR), `release` (manual only)
- **Outputs**: Single APK — `mobile-arm64_v8a`（手机版 ARM64）
- **Artifacts**: Uploaded as `TV-{build_type}` to the workflow run

### Required secrets (for release builds)

| Secret | Purpose |
|--------|---------|
| `STORE_PASSWORD` | Keystore & key password |
| `KEY_ALIAS` | Signing key alias |
| `KEY_PASSWORD` | Key password (falls back to `STORE_PASSWORD`) |

For **debug builds**, no secrets are needed — the workflow auto-generates a throwaway debug keystore.

### Note on compileSdk 37

The project targets `compileSdk = 37`, which is a preview/future API level. If the SDK manager cannot install it, adjust `env.COMPILE_SDK` in the workflow or downgrade `compileSdk` in `gradle/libs.versions.toml` to a stable SDK level.
